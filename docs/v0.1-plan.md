# v0.1 Plan

## Goal

Deliver the smallest credible end-to-end reference implementation in roughly 2 weeks of focused work.

## Success criteria

- `docker compose up` starts Postgres, Redis, and the MCP server
- small synthetic dataset loads successfully
- `get_inventory`, `source_order`, and `explain_decision` work
- single-line order sourcing is deterministic and testable
- inventory reservation is idempotent
- decision events are persisted and inspectable
- one notebook or scripted demo proves the full flow

## Explicit scope for v0.1

### Include
- Python 3.11+
- FastMCP server
- Pydantic models
- Postgres persistence
- Redis for idempotency and hot inventory reads
- OR-Tools solver for single-line sourcing
- deterministic promise dates
- append-only event logging
- seeded small Hearthline-like dataset

### Exclude
- multi-line split-shipment optimization
- conformal prediction
- LangGraph exception workflow
- policy hot reload
- replay CLI
- benchmark harness
- full 80k SKU dataset

## Recommended milestone sequence

### Milestone 1 — Repository skeleton
- pyproject
- package layout
- Docker Compose
- environment sample
- lint/test tooling

### Milestone 2 — Domain and storage
- domain models
- database schema
- repositories
- idempotency store
- inventory seed load

### Milestone 3 — Deterministic kernel
- candidate node filtering
- single-line sourcing solver
- deterministic promise-date function
- reservation transaction flow

### Milestone 4 — MCP surface
- `get_inventory`
- `source_order`
- `explain_decision`
- OpenTelemetry spans

### Milestone 5 — Demo and docs
- demo notebook or script
- quickstart README
- architecture walkthrough
- Claude Desktop MCP screenshot later

## Primary risks

1. Over-scoping the first version
2. Spending too long on synthetic data realism
3. Trying to ship the LLM exception path too early
4. Over-designing the policy DSL before real use cases exist
5. Combining explanation generation too tightly with commit flow

## Recommendation

Keep v0.1 brutally narrow. The goal is not to look complete; it is to look real.
